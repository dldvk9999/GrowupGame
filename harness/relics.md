# 유물(Relic) 시스템 (migration 119/120, 신규 콘텐츠, 사용자 요청)

관련 파일: `RelicGacha.jsx`, `lib/relicCatalog.js`, `lib/relicGacha.js`, `lib/relicBonus.js`, migration 119/120

## 개요

스킬 뽑기와 동일한 단일 풀 구조(전역 뽑기레벨, `profiles.total_relic_draws`)로 50종 유물을 뽑는 콘텐츠. 장비와 다른 점은:
- **중복 = 강화 시도**(장비는 100% 자동 강화, 유물은 실패할 수 있음)
- **최대 3개까지만 장착**해야 효과가 적용됨(장비/스킬은 보유만 해도 소량의 "보유효과"가 있는데, 유물은 그게 없음 — 이유는 아래 "밸런스 설계" 참고)

## 유물 카탈로그 (50종)

효과 카테고리 7종 × 수치(flat)/비율(percent) 방식을 섞어 10개 원형(archetype)을 만들고, 5개 등급에 각각 10개씩 배치:

| 카테고리 | flat | percent |
|---|---|---|
| 체력(hp) | ✅ | ✅ |
| 공격력(atk) | ✅ | ✅ |
| 방어력(def) | ✅ | ✅ |
| 스킬 쿨타임(cooldown) | — | ✅ |
| 골드 획득(gold) | — | ✅ |
| 경험치 획득(exp) | — | ✅ |
| 버프 효과(buff) | — | ✅ |

체력/공격력/방어력은 "고정 수치로 늘어나는 것"과 "비율로 늘어나는 것" 둘 다 존재(사용자 요청 반영)하도록 flat+percent 둘 다 만듦. 쿨타임/골드/경험치/버프는 원래 성격상 비율(%) 단위가 자연스러워서(예: "쿨타임 -0.6%") percent만 만듦 — flat 버전(예: "쿨타임 -30ms")은 스킬마다 쿨타임 격차가 800ms~22000ms로 워낙 커서 의미가 옅어질 것 같아 제외.

등급 배율은 장비 등급 배율(`itemCatalog.js RARITIES`)과 동일하게 1 / 1.8 / 2.8 / 4.2 / 6.5 — 다른 시스템과 감각적으로 통일.

## 강화 (최대 +200, 실패 확률 있음)

중복 유물을 뽑으면 **강화 시도**가 발생함:
```
성공확률 = max(0.30, 1 - 현재레벨 × 0.0035)
```
- 레벨 0→1(첫 중복): 100%
- 레벨 50: 82.5% / 레벨 100: 65% / 레벨 150: 47.5% / 레벨 199: 최저 30%(바닥)

**실패해도 레벨이 깎이지 않음** — 이 게임의 전반적인 "덜 가혹한" 톤(`ui-and-ux.md`의 디자인 완화 이력 참고)을 유지하기 위한 의도적 선택. 그냥 이번 중복 하나가 강화에 실패하고 사라질 뿐, 다음에 또 뽑으면 됨. 30% 바닥을 둬서 "사실상 불가능"해지는 구간이 없도록 함.

효과치 공식(서버 `calc_relic_bonus`/클라 `getRelicEffectiveValue`, 반드시 동일하게 유지):
```
효과치 = base_value × rarity_mult × (1 + level × 0.03)
```
+3%/레벨로 최대 200레벨이면 최종 7배(장비의 +8%/레벨×1000레벨=81배보다 훨씬 보수적으로 잡음 — 아래 밸런스 설계 참고).

## 밸런스 설계 (사용자 요청: "밸런스가 너무 튀지 않게")

**핵심 안전장치는 "최대 3개 장착 제한"**. 만약 보유한 유물 전부(최대 50개)가 장비처럼 "보유효과"로 항상 적용됐다면, 후반부에 유저가 유물을 많이 모을수록 파워가 무한정 누적되는 구조가 되어 기존 장비/스킬 시스템을 압도할 위험이 컸음. 대신 "장착한 3개만" 적용되게 해서 총 파워를 구조적으로 제한함 — 장비(4슬롯 고정 카테고리)와 달리 유물은 50종 중 자유 선택이라 "어떤 3개를 낄지"라는 전략적 재미는 남기면서도 파워 총량은 항상 최대 3개 몫으로 묶여있음.

극단적 계산(신화 등급 + 200강, ATK% 유물 3개를 전부 장착): 3 × (0.8% × 6.5 × 7) = 약 109% ATK 증가. 이 정도 수준에 도달하려면 신화 등급 ATK% 유물을 3개나 뽑아야 하고(50종 중 신화 ATK%는 딱 1종뿐이라 애초에 3개 장착 자체가 불가능 — 실제로는 서로 다른 카테고리를 섞어 낄 수밖에 없음), 거기에 각각 200강까지(성공확률이 갈수록 낮아지는) 강화해야 하는 극단적 엔드게임 투자가 필요함. 일반적인 유저(에픽~전설 등급, 낮은~중간 강화)는 한 자릿수 % 수준의 보너스만 받게 되어 기존 시스템을 압도하지 않는 "보조 성장 수단"으로 설계됨.

## 적용 범위 (의도적으로 제한된 부분)

전투가 부분적으로 클라이언트에서 계산되는 이 게임 구조상(`security.md`의 "알려진 한계" 참고), 유물 효과의 적용 범위를 다음과 같이 명시적으로 제한함:

- **체력/공격력/방어력(flat+percent)**: 서버(PvP 전투력·랭킹) + 클라이언트(전투 화면) 양쪽 다 반영. `App.jsx`의 `equipmentBonus` 객체에 유물 몫을 합산해서 넘기므로, 전투/던전/전직던전/월드보스 4개 전투화면 **전부** 자동으로 적용됨(화면별로 따로 손댈 필요 없음 — 기존 장비 보너스와 같은 파이프라인을 타기 때문)
- **골드 획득%**: **자동사냥(`grant_idle_reward`)에만** 반영 — 105(주말보너스)/062(황금몬스터)와 동일한 범위 제한 원칙. 스테이지 클리어/던전/탑/월드보스 개별 보상 함수는 범위 밖(건드리지 않음)
- **경험치 획득%**: `BattleScreen.jsx`의 **자동사냥 경험치에만** 반영(골드와 동일한 범위 제한 논리) — 스테이지 도전(챌린지 모드) 클리어 경험치는 범위 밖
- **스킬 쿨타임 감소%/버프 효과%**: **`BattleScreen.jsx`(일반 전투 화면)에만** 반영 — 던전/전직던전/월드보스 전투화면은 이번 1차 범위에 포함 안 함(4개 화면 전부 손대기엔 회귀 리스크 대비 이득이 낮다고 판단, 필요하면 추후 확장). 쿨타임 감소는 최대 90%까지, 버프 강화로 인한 haste 감소도 최대 95%까지 안전 클램프(0 이하로 내려가 타이머가 깨지는 것 방지)

## 뽑기 UI (`RelicGacha.jsx`)

상점의 6번째 탭 "🏺 유물 뽑기" — 스킬/장비 뽑기와 달리 **무료뽑기 대상이 아님**(`daily_free_draw_state`의 `draw_type` 체크 제약에 `relic`이 없음, 의도적으로 제외). 뽑기 결과 카드에 강화 성공/실패를 색으로 구분 표시(성공 초록 계열/실패 빨강 계열), 결과 공유 버튼도 동일 패턴으로 지원. 화면 하단에 보유 유물 목록(등급 내림차순 정렬)이 카드 그리드로 뜨고, 카드를 탭하면 장착 선택(최대 3개, 초과 시 토스트로 안내) → "장착 저장하기" 버튼으로 서버에 반영.

## 알려진 제약사항 / 후속 검토 항목

- PvP 상대의 유물 보너스도 서버에서 실시간 계산되므로(계산 비용 증가), 랭킹/매칭 조회가 살짝 느려질 수 있음(체감상 무시할 수준으로 판단, 문제되면 캐싱 검토)
- 던전/전직던전/월드보스 전투화면엔 쿨타임감소%/버프% 미반영(위 "적용 범위" 참고) — 사용자 반응 보고 확장 검토
- 유물 획득 관련 업적(예: "유물 N종 수집", "유물 200강 달성")은 아직 없음 — 다른 콘텐츠와 마찬가지로 나중에 추가 검토 가능
