# 월드보스

관련 파일: `WorldBossBattle.jsx`, `DungeonSelect.jsx`(월드보스 탭), `worldBoss.js`, migration 033

## 개요

전체 유저가 **공유 체력풀 하나**를 함께 깎는 주간 레이드 콘텐츠. 던전 탭의 "🐉 월드보스" 서브탭에서 접근.

- **체력은 매주 일요일 00:00(서울시간)에 초기화**됨 — 그 전에 다 못 깎으면 새로운 개체로 리셋
- 하루 **3회**까지 도전 가능(다른 던전과 동일하게 오전 8시 기준 초기화)
- **한 판당 제한시간 1분** — 시간 안에 못 잡아도 그동안 입힌 피해는 그대로 공유 체력에 반영됨(퇴각/시간초과 모두 "그때까지 입힌 데미지"를 서버에 보고하는 동일한 경로를 탐)
- 체력/공격력이 매우 높게 잡혀있어서(아래 참고), **4차 전직 정도는 해야 유효타가 들어감**
- 클리어(체력 0)하면 **그 주에 조금이라도 피해를 입힌 유저 전원**에게 7일짜리 "용의 버프" 지급 + 헤더 닉네임 그라데이션 하이라이트 + **피해량 비례 골드 보상(우편함)**
- 못 잡고 주가 끝나면, 각자 **누적 피해량에 비례한 골드**를 우편으로 보상

## 보스 스탯

`world_boss_state` 테이블에 저장, 매주 새로 생성될 때 고정값으로 세팅됨:

| 항목 | 값 |
|---|---|
| 최대 체력 | 30,000,000 |
| 공격력 | 7,000 (035에서 4,500→7,000으로 상향, 진행 중인 주에도 즉시 반영됨) |
| 방어력 | 1,200 |

⚠️ 정확한 밸런스는 실제 플레이해보고 조정이 필요함(다른 신규 콘텐츠와 동일한 원칙) — 특히 유저 수/평균 전투력에 따라 "일주일 안에 다 같이 깎을 수 있는지"가 크게 달라짐. `sync_world_boss()` RPC의 `insert into world_boss_state (...)` 값만 바꾸면 됨.

## 전투 화면 - "압도적인" 비주얼

일반 전투화면(스테이지/던전)은 적/플레이어가 비슷한 크기로 좌우 대칭 배치되지만, 월드보스는 **전용 아레나 레이아웃**(`.worldboss-arena`)을 씀:

- 보스 아이콘(🐉)이 **화면 전체를 채우는 크기**로 중앙에 표시됨 — `clamp(290px, 34vw, 340px)`(데스크톱), 기존 버전의 월드보스 아이콘(72px) 대비 **4배 이상** 큼. 모바일에서는 레이아웃이 깨지지 않도록 `clamp(160px, 62vw, 260px)`로 별도 축소되지만, 그래도 원래보다 훨씬 큼
- 은은하게 커졌다 작아지는 `worldboss-loom` 애니메이션 + 보라/주황 이중 그림자(`drop-shadow`)로 위압감 연출
- 플레이어는 기존과 동일하게 작은 크기로 한쪽 구석에 배치 — "나 vs 압도적으로 거대한 것"이라는 크기 대비를 의도적으로 강조

## 전투 방식

`WorldBossBattle.jsx`는 `DungeonBattle.jsx`와 거의 동일한 전투 엔진(스킬 타입별 로직, 쿨타임 UI 등은 [`combat.md`](./combat.md)/[`skills.md`](./skills.md) 참고)을 쓰지만 다음이 다름:

- 몬스터 도감 성장(exp)은 없음 — 순수하게 "이번 전투에서 보스에게 얼마나 피해를 입혔는가"만 추적(`damageDealtRef`)
- 전투는 플레이어가 죽거나(퇴각), 보스가 죽을 때(그 판에서 마지막 일격을 넣었을 때, 드묾), 또는 **1분 제한시간이 다 될 때(시간초과)**까지 진행 — 셋 다 결과값만 다를 뿐(`'win'`/`'lose'`/`'timeout'`) 종료 시 서버 보고 경로는 동일함
- 전투 종료 시 `report_world_boss_damage(weekKey, damageDealt)` RPC로 **실제 입힌 데미지를 서버에 보고**하고, 서버가 계산한 공식 결과(반영 후 남은 체력, 클리어 여부)를 받아서 화면에 반영

### 서버 검증 (안티치트)

전투 자체는 다른 화면들과 마찬가지로 클라이언트에서 계산되므로([`security.md`](./security.md)의 "알려진 한계" 참고), 보고된 데미지를 무조건 믿지 않고 **전투력 기준 상한으로 클램프**함:

```
1회 전투 최대 인정 데미지 = max(2000, 내_전투력 * 60)
실제 반영 데미지 = min(보고된_데미지, 위_상한, 보스_남은체력)
```

`fetch_my_combat_power()`(PvP와 동일한 헬퍼)로 서버가 직접 재계산한 전투력을 기준으로 삼기 때문에, 클라이언트가 데미지 숫자를 조작해서 보내도 상한 이상은 반영되지 않음.

## 클리어 보상 — 용의 버프

체력이 0 이하가 되면:

1. `world_boss_state.cleared = true`로 마감
2. 그 주 `world_boss_contributions`에 피해량이 1 이상 기록된 **모든 유저**의 `profiles.dragon_buff_until`을 `now() + 7일`로 설정 (즉시 적용, 버프는 우편 개념이 아니라서 이 부분만 예외)
3. **같은 유저들에게 피해량 비례 골드 보상도 우편으로 발송**(`reward = min(500000, max(300, round(누적피해량/100)))`, `source_key`로 중복지급 방지) — 미클리어 보상(아래)보다 환산율이 후함
4. 이후 7일간 로그인할 때마다 `App.jsx`가 `dragon_buff_until > now()`인지 확인해서:
   - **공격력/방어력을 "지금의" 20배로 만듦** — 장비 보유효과/스킬 보유효과까지 다 반영된 현재 유효 수치를 기준으로 20배가 되도록, `equipmentBonus`에 필요한 만큼(기존 유효수치의 19배)을 추가로 얹는 방식으로 구현(`App.jsx`). 원본 `owned_monsters`의 저장된 스탯 자체는 건드리지 않음(버프가 끝나면 자동으로 원래대로 돌아옴)
   - **헤더 닉네임에 무지개 그라데이션 애니메이션** 적용(`.app-nickname--dragon`, `index.css`)

## 미클리어 보상

`sync_world_boss()`가 새 주의 보스를 만들기 직전에, **직전 주가 클리어되지 않았다면** 그 주 `world_boss_contributions`를 순회하며 각 유저에게:

```
보상 골드 = min(300000, max(100, round(누적_피해량 / 150)))
```

`add_gold` 내부 호출 + 우편함으로 안내 메일 발송(`source_key`로 중복 지급 방지).

## 지연 생성 패턴

우편함/PvP상점과 동일하게 **cron 없이 지연 생성**됨 — `sync_world_boss()`가 월드보스 화면 진입 시 호출되어, "이번 주(일요일 기준) 보스가 아직 없으면 그때 생성 + 지난 주 정산"을 처리함. 아무도 접속하지 않는 주가 있으면 그 사이 정산이 미뤄질 수 있음(다음 접속자가 트리거).

주차 키(`week_key`)는 일요일 날짜(YYYY-MM-DD) 문자열이고, `date_trunc('week', ...)`가 기본적으로 월요일 기준이라 `+1일 후 truncate, -1일`로 일요일 기준으로 보정해서 계산함.
