# UI / UX 공통

관련 파일: `toast.js`, `ToastContainer.jsx`, `usePwaInstall.js`, `index.css`

## 토스트 알림

- 간단한 pub-sub(`showToast(message, type)`) + `ToastContainer`가 구독해 화면 상단 중앙에 표시(3.2초 후 자동 소멸)
- 골드 부족 에러가 발생하는 모든 지점에서 공통으로 토스트 사용 — 새 골드 소비 기능 추가 시도 동일 패턴 유지할 것

### ⚠️ 버튼 disabled 함정 주의

구매/강화/뽑기/던전입장 버튼을 "실행 불가 시 `disabled`"로 막으면 클릭 자체가 안 돼서 에러 발생 기회가 없어지고 토스트도 못 뜸(실제 있었던 버그, 수정됨).

골드 소비/던전 입장 버튼은 **클릭은 항상 가능하게 두고**, 핸들러 진입 시 조건 체크 후 `showToast` + 조기 return. 버튼의 "비활성처럼 보이게"는 `disabled` 대신 `.btn-unaffordable`(투명도+빨간 테두리) CSS로만 처리. 서버가 그래도 거부하는 경우(레이스컨디션 등)를 대비해 `App.jsx` catch에서도 동일하게 `showToast`(이중 방어).

## 키보드 단축키

각 화면 컴포넌트가 자기 스코프 안에서만 `window.addEventListener('keydown', ...)`를 등록/해제(전역 단일 핸들러 아님). 입력창 포커스 시엔 전부 무시.

| 화면 | 키 | 동작 |
|---|---|---|
| 전투(스테이지/일반던전/전직던전) | `1`~`9` | 슬롯별 스킬 즉시 사용 — 슬롯 최대치가 10이어도 단축키는 9개까지만, 10번째는 클릭만 |
| 스테이지 전투 | `Space` | 상황별: 자동사냥 중→도전 시작 / 승리→다음 스테이지로 / 패배→재도전 |
| 스테이지 전투 | `R` | 패배 시 즉시 재도전 |
| 일반/전직 던전 전투 | `Space` | 결과 화면에서 "던전 목록으로"(전직 성공 시 "확인") |
| 상점(뽑기 4탭) | `Tab` / `Shift+Tab` | 무기→방어구→장갑→신발→스킬 순환(스킬편성은 최상위 탭이라 제외) |
| 던전 탭 | `Tab` / `Shift+Tab` | 경험치→골드→전직 던전 순환 |
| 장비/스킬 뽑기 화면 | `G` / `Shift+G` / `Ctrl(⌘)+G` | 1회 / 10회 / 100회 뽑기 |
| 설정 > 우편함 | `Enter` | 미수령 우편 전체 일괄수령 |
| 설정 > 쿠폰 입력 | `Enter` | 입력창 포커스 상태에서 폼 제출(HTML form 기본 동작) |
| 마이페이지/설정 화면 | `Esc` | 전투 탭으로 복귀 |

`Space`/`Tab`은 브라우저 기본 동작과 충돌해 `e.preventDefault()` 처리. 배틀 화면 3곳은 상태가 자주 바뀌어 클로저 문제를 피하려고 `useRef`에 최신 상태를 매 렌더 담고 `keydown` 리스너는 `useEffect(..., [])`로 1회만 등록.

## 모바일 레이아웃 겹침 주의

가이드 미션 플로팅 버튼이 `position:fixed`라 콘텐츠 하단(특히 전투 스킬버튼 줄)을 가릴 수 있음 — `app-main` 하단 padding을 데스크톱 110px/모바일 150px로 넉넉하게 잡아 방지(`index.css`). **하단 새 UI 추가 시 이 여백 고려할 것.**

### flex 가로 카드형 UI의 모바일 안전 패턴

아이콘+정보+버튼을 한 줄에 나열하는 flex 카드는 버튼 텍스트가 길어질 경우를 대비해 **`flex-wrap: wrap`을 기본으로 걸어둘 것** — 안 들어가면 가로 스크롤 대신 다음 줄로 넘어감. `flex:1`인 정보 영역엔 `min-width:0` 대신 **적당한 하한값(예: 140px)**을 줘서 가독성 유지(업적 화면, 050 적용).

## 과거에 있었던 CSS 스크롤 버그

`html, body, #root`에 `height: 100%`(고정값)면 콘텐츠가 뷰포트보다 길어질 때(상점 화면 등) 하단이 잘리는 문제. `min-height: 100%`로 해결 — "하단 짤림" 재발 시 이 부분부터 의심할 것.

### `overflow-x: hidden`을 걸었다가 `position: sticky`가 전부 깨졌던 회귀

모바일 가로스크롤을 막으려 `html, body, #root`에 전역 `overflow-x: hidden`을 추가했더니, `.loadout-sticky-bar` 등 `position: sticky` 요소가 전부 깨짐. **overflow를 지정한 조상이 있으면 그게 sticky의 스크롤 컨테이너가 되어 뷰포트 기준 sticky가 더 이상 안 붙음**(CSS sticky의 잘 알려진 함정).

→ **수정**: 전역 `overflow-x: hidden`을 제거. 실제 가로스크롤이 필요한 곳(`.tab-nav`, `.chapter-carousel`)은 이미 로컬로 `overflow-x: auto` + 스크롤바 숨김이 걸려있어 문제없음.

**교훈**: 가로스크롤을 막고 싶다고 `html`/`body`/`#root`에 전역 `overflow-x: hidden`을 거는 건 `position: sticky` 요소를 전부 깨뜨리는 광범위한 부작용이 있음. 실제 오버플로우가 발생하는 구체적 요소에 국소적으로 적용하는 게 안전함.

## 전반적인 디자인 톤 완화 (딱딱함 개선)

사용자 피드백으로 "너무 딱딱하다"는 지적을 받아, 구조는 안 건드리고 시각 속성 위주로 다듬음:

- **디자인 토큰 확장**(`:root`): `--radius-md` 10px→14px, `--radius-lg` 16px→20px 상향(전역 변수라 자동 반영). `--radius-sm`(10px), `--shadow-soft`/`--shadow-lift`, `--ease-soft`(부드러운 이징) 신규
- **버튼(`.btn`)**: box-shadow 추가, hover 시 살짝 떠오름(`translateY(-1px)` + 밝기 증가) — 이전엔 `:active`만 반응해 데스크톱에서 밋밋했음
- **카드류**: `box-shadow: var(--shadow-soft)` 일괄 추가, 로그인/스토리 카드는 대각선 그라데이션도 추가
- **탭 네비게이션**: active 상태를 단색 대신 그라데이션+그림자, transition 추가
- **리스트 행**: hover 시 테두리 색 전환 추가
- **진행바**: 완전히 둥근 캡슐형(`border-radius: 999px`) + 안쪽 그림자 + 느긋한 채워짐 이징
- **인풋 필드**: 포커스 시 딱딱한 아웃라인 대신 부드러운 glow(box-shadow 링)
- **모달**: 배경 페이드인 + 패널이 살짝 아래에서 떠오르는 등장 애니메이션(`modal-pop-in`)
- **미확인 알림 뱃지**: 은은한 pulse 애니메이션 추가

전부 기존 클래스명/구조 유지, 시각 속성만 조정이라 회귀 위험 낮음 — 빌드 후 CSS 중괄호 균형 검증 + Playwright 정적 프리뷰로 그라데이션 색상 픽셀 단위 확인.

## 모바일 인벤토리 가로 넘침 버그 수정 (사용자 제보)

**증상**: 모바일 인벤토리 화면이 옆으로 밀리고 텍스트가 넘어감.

**원인**: `.inventory-row`가 `display: flex`인데 `flex-wrap` 미지정(기본 `nowrap`)이라, 아이콘+이름/강화진행바+스탯 2줄+액션버튼 2개를 전부 한 줄에 욱여넣으려 함. 필요 폭 합이 좁은 화면(360~380px)에서 초과해 가로 스크롤 발생.

**검증**: Playwright로 380px에서 `scrollWidth` 측정 — 수정 전 395px(15px 초과), 수정 후 380px(정확히 일치).

**수정**: `.inventory-row`에 `flex-wrap: wrap`, `.inventory-row-actions`에 `margin-left: auto`(새 줄에서도 우측 정렬), `.inventory-name-col`을 `flex:1 1 120px`로, `.inventory-stat-group`에 `min-width:0; flex-shrink:1;`(flex 기본 `min-width:auto` 특성 때문에 필요).

**교훈**: `display: flex`로 여러 텍스트/버튼 그룹을 배치하는 카드는 처음부터 `flex-wrap: wrap`을 기본값으로 걸어둘 것 — `Inventory.jsx`는 이 패턴이 확립되기 전(초기 세션)에 만들어져서 놓쳤음, 오래된 화면도 주기적 재점검이 필요함.

## 유사 버그 전수 재스캔 (inventory-row 수정 계기)

`display: flex` + `align-items: center` + `flex-wrap` 미지정 패턴 전체를 정규식으로 스캔(27개 후보). 실제 카드형 row만 추려 380px + 긴 텍스트 실측 결과 전부 안전: `.mail-row`(구조적으로 안전), `.mypage-row`(자연 줄바꿈), `.loadout-slot`(고정 정사각형), `.worldboss-contributor-row`(이미 ellipsis 처리), `.inventory-main`(죽은 클래스).

**결론**: `inventory-row`가 유일한 실제 버그였음.

## 우편함 미수령 버튼 텍스트 줄바꿈 버그 수정 (사용자 제보)

**증상**: 우편함 미수령 우편의 제목/본문이 부자연스럽게 줄바꿈됨.

**원인**: 미수령 "수령" 버튼이 `.btn-challenge`(`width: 100%`)를 그대로 씀. `.mail-row`가 flex라 버튼의 `width:100%`가 flex-basis 계산을 꼬아, 340px 행에서 버튼이 156px(절반)를 차지하고 `.mail-row-main`이 137px로 짜부러들어 원래 한 줄에 들어갈 제목까지 여러 줄로 쪼개짐.

**검증**: 수정 전 `mail-row-main` 137px/버튼 156px → 수정 후 218px/76px.

**수정**: 미수령 버튼에 전용 클래스 `mail-claim-btn` 추가, `.mail-claim-btn { width: auto; max-width: none; flex-shrink: 0; }`로 오버라이드. `.mail-row-main`에도 `flex:1; min-width:0;` 명시.

**교훈**: `width:100%`가 있는 범용 버튼 클래스를 flex row 안에서 다른 요소와 나란히 쓸 때는 항상 주의 필요 — 새로 만들 때 버튼에 `flex-shrink:0`과 `width:auto` 오버라이드를 세트로 붙이는 습관 필요.

## 탭 전환/백그라운드 복귀 시 마지막 탭이 유지 안 되던 버그 수정 (사용자 제보)

**증상**: 다른 탭/앱을 봤다가 돌아오면 항상 "⚔️ 전투" 탭으로 되돌아감.

**원인**: `handleSession(newSession)`을 등록하는 `useEffect`가 **빈 의존성 배열**이라 최초 마운트 시 1번만 실행됨. 그런데 세션 갱신(토큰 리프레시, 백그라운드 복귀 등)마다 다시 호출되는데, 그때마다 **무조건 전체 데이터를 재조회하고 `activeTab`을 `'battle'`로 강제 리셋**하고 있었음.

**수정**: `sessionRef`(useRef)로 마지막 세션의 유저 ID를 추적해 "같은 유저의 세션 갱신인지" 판별. 같은 유저 재확인이면 `session` state만 갱신하고 조기 리턴해 데이터 재조회/탭 리셋을 생략. 최초 로그인/계정 전환은 기존처럼 전체 초기화됨.

**교훈**: 한 번만 등록되는 리스너 콜백 안에서 최신 state를 직접 참조하면 "최초 렌더 시점의 오래된 값"만 보게 되는 클로저 함정. "새 이벤트인지 재확인인지" 구분이 필요하면 state 대신 `useRef`로 최신값을 추적할 것.

**부수 효과 — 가이드미션 재완료 표시 버그도 함께 해결됨**: `hasClaimedMissionToday`도 세션 갱신마다 함께 리셋되고 있었는데, 이 수정으로 자동 해결됨.

## 넓은 화면(PC 브라우저)에서 주요 액션 버튼이 왼쪽으로 치우치던 버그 수정 (사용자 제보)

**증상**: "오늘의 무료뽑기", "⚔️ 대결 시작" 같은 버튼이 중앙정렬이 안 됨.

**원인**: 모바일 우선 설계로 지금까지 360px 뷰포트로만 검증했는데, `.btn-challenge`가 `width:100%; max-width:360px`라 **넓은 화면에서는 부모(`.game-shell`, 최대 860px)보다 버튼이 좁아지며 왼쪽에 붙음**. 모바일에선 `max-width`가 실질적으로 안 걸려 항상 꽉 차 보였음.

**수정**: `.btn-challenge`에 `display:block; margin-left:auto; margin-right:auto;`로 항상 수평 중앙. 단 **다른 버튼과 나란히(flex row) 배치되는 곳**(`.result-actions`, `.gacha-draw-buttons`, `.lobby-chat-form`)에선 `margin:0; width:auto;`로 되돌림. 이 과정에서 **채팅 전송 버튼도 같은 이유로 넓은 화면에서 360px까지 부풀던 것을 추가 발견/수정**.

**검증**: Playwright로 1200~1400px에서 버튼 중심과 콘텐츠 영역 중심 일치 확인, 4개 컨텍스트 모두 모바일(360px) 회귀 없음 재검증. **교훈**: 모바일 우선 검증만으로는 넓은 화면 문제를 놓칠 수 있음 — `width:100%; max-width:Npx` 패턴은 넓은 뷰포트도 별도 검증할 것.

## 전 화면 반응형(모바일+넓은화면) 전수조사 (사용자 요청)

**검증 방법**: (1) `max-width` 선언 CSS 규칙 전수 나열해 `margin:auto`/부모 flex 중앙정렬 확인, (2) `App.jsx` 화면 구조 추적해 모든 게임 화면이 예외 없이 `.game-shell`(`max-width:860px; margin:0 auto`) 안에서 렌더링됨을 확인, (3) 로그인 전 화면은 `.center-viewport`, 모달은 `.modal-backdrop`, 에러 화면은 `.error-boundary-screen`이 각각 flex 중앙정렬로 감싸고 있음을 확인 — 즉 4개 "최상위 컨테이너"만 정확히 중앙정렬되면 개별 화면은 `width:100%`(상대적)만 신경 쓰면 안전함.

**결과**: 이미 고친 것 외 추가 문제 없음(인라인 `style={{maxWidth}}` 사용처도 전무). `position:fixed` 요소들(토스트, 미션 플로팅 버튼)은 의도적으로 중앙정렬 아니므로 정상. **새 화면 추가 시 이 4개 셸 중 하나(대부분 `.game-shell`) 안에 넣기만 하면 반응형이 자동 보장됨을 항상 기억할 것.**

## 전투화면 스킬 영역 반응형 통일 (사용자 요청)

**배경**: 월드보스 스킬 최대 10개(전직 5차 기준) 대응을 위해 만들었던 `worldboss-skills-row`(항상 46×46px 아이콘 전용 압축)가 **PC에서도 무조건 적용**돼있었음. 반대로 전투/던전/전직던전 화면은 기본 `.skill-btn`(이름 노출, 90px+)이 **모바일에서도 그대로**라 스킬이 많아지면 화면 아래가 길어짐. 사용자가 "월드보스는 PC에서도 다른 화면처럼 이름 노출+넓게, 다른 화면은 모바일에서 월드보스처럼 압축"으로 통일해달라고 요청.

**수정**: 화면별 전용 클래스를 없애고 **화면 폭 기준 반응형 하나로 통일**:
- 기본(`.skill-btn`, `@media (min-width:1024px)` 포함): 아이콘+이름 노출, `flex:1 1 100px` 이상 — PC/태블릿은 4개 전투화면(전투/던전/전직던전/월드보스) 모두 동일
- `@media (max-width:640px)`: `.skill-btn`을 46×46px 정사각형으로 압축, `.skill-name { display:none }` — 모바일은 4개 전투화면 모두 동일하게 아이콘+쿨다운링+단축키만
- `WorldBossBattle.jsx`가 쓰던 `worldboss-skills-row` 클래스는 제거하고 다른 화면과 동일한 `.skills-row`만 사용 — CSS 규칙이 화면 폭만으로 결정되므로 컴포넌트별 분기가 필요 없어짐(유지보수 단순화)
- `SkillButton.jsx`는 애초에 아이콘/이름을 항상 함께 렌더링하는 컴포넌트라 JSX 변경 없이 **CSS만으로** 압축/확장이 전환됨(이름 유무를 `display:none`으로 토글) — 이미 월드보스에서 검증된 패턴이라 리스크 낮음
