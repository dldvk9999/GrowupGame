# BGM/효과음 (신규 콘텐츠, `lib/audio.js`)

## 개요

실제 음원 파일 없이 **Web Audio API의 오실레이터로 직접 합성**한 8비트풍 신스 사운드. 외부 에셋 로딩이 없어서 가볍고 네트워크 상태와 무관하게 항상 작동함.

## 구성

- **효과음(SFX)**: 공격/힐/버프/승리/레벨업/골드/황금몬스터/신기록/뽑기결과(등급별 음색 차등) — 짧은 오실레이터 톤 조합
- **BGM**: 5음계(펜타토닉) 기반 짧은 루프 멜로디를 `setTimeout`으로 계속 이어붙여서 재생. 진짜 음원이 아니라 순차 합성이라 "찬찬한 신스 루프" 느낌

## 브라우저 자동재생 정책 준수

브라우저는 사용자 상호작용 없는 오디오 자동재생을 막음. `App.jsx`가 `document`에 **1회성 클릭 리스너**를 등록해서, 화면 어디든 처음 클릭하면(BGM 설정이 켜져있는 경우) 그 안에서 `AudioContext`를 초기화/resume하고 BGM 시작 — 이후 리스너는 스스로 해제됨.

## 설정 (게임가이드 화면)

BGM on/off, 효과음 on/off, 볼륨 슬라이더를 설정 > 게임가이드 상단에 배치(테마 컬러 선택과 같은 "개인화 설정" 묶음). `localStorage`에 저장, 서버 관여 없음.

- ⚠️ **BGM 기본값 변경(신규, 사용자 요청)**: 최초 접속(저장된 설정이 아예 없는 상태) 시 `bgmEnabled` 기본값을 `true`→`false`로 변경 — 효과음(`sfxEnabled`)은 그대로 기본 켜짐 유지. `startBgm()`이 이미 `settings.bgmEnabled`를 체크하고 무음이면 그냥 return하는 구조라, `loadSettings()`의 기본값 하나만 바꾸면 충분했음(다른 곳을 별도로 손댈 필요 없음). 기존에 이미 BGM을 켜둔 유저는 `localStorage`에 저장된 값이 우선이라 영향 없음.

## 통합 지점

- **전투(`BattleScreen.jsx`)**: 데미지/힐/버프 스킬 사용 시 각각 다른 효과음, 승리 시 팡파레, 레벨업 시 딜레이 후 사운드
- **뽑기(`SkillGacha.jsx`, `EquipmentGacha.jsx`)**: 결과 중 최고 등급 기준으로 음색 차등(등급 높을수록 화음 풍부)
- **황금 몬스터 이벤트**: 반짝이는 상승 아르페지오
- **무한의 탑 신기록**: 팡파레풍 사운드
- **로그인 시 골드 보상(오프라인/복귀, 102~105)**: `playGoldSound()` 재사용 — 토스트가 뜨는 타이밍(순차 표시, 위 [`mailbox-and-coupons.md`](./mailbox-and-coupons.md)/[`stages-and-dungeons.md`](./stages-and-dungeons.md) 참고)과 정확히 맞춰서 재생되도록 `setTimeout` 콜백 안에서 같이 호출함(사운드가 토스트보다 먼저 울리면 어색해서 동기화 필요했음). 안내성 토스트(오늘의한마디/주말이벤트)는 사운드 없음

## ⚠️ 배포 전 발견/수정한 버그: 첫 사운드 재생 실패

Playwright로 각 함수를 헤드리스 브라우저에서 실제 호출해 검증하던 중 발견: **모듈 로드 후 가장 처음 호출되는 사운드 함수가 항상 에러를 냄**(`Failed to execute 'connect' on 'AudioNode': Overload resolution failed`).

**원인**: `sfx()`가 `tone(freq, duration, type, sfxGain, ...)`를 호출할 때 `sfxGain`은 호출 시점에 이미 평가되어 고정된 값으로 전달됨(JS 인자 평가 규칙). 그런데 `sfxGain`은 `tone()` *내부*에서 `getContext()` 호출 시 비로소 초기화됨 — 즉 최초 호출 땐 아직 `null`인 값이 그대로 박제 전달되고, 전역 `sfxGain`은 갱신돼도 이미 넘어온 매개변수는 여전히 `null`이라 `env.connect(null)`이 실패함. 두 번째 호출부터는 전역이 이미 초기화돼있어 재현 안 됨(순차 여러 함수 호출 테스트가 아니었으면 놓쳤을 버그).

**수정**: `sfx()`에서 `tone()` 호출 **전에** `getContext()`를 먼저 명시적으로 호출해서 순서를 바꿈. `scheduleBgmNote()`는 이미 같은 순서라 이 버그가 없었음.

**검증**: 로컬 HTTP 서버 + Playwright로 `audio.js`의 모든 export(16개 시나리오: 효과음 10종 + 뽑기 등급별 5종 + BGM 시작/정지)를 실제 호출해 예외 없이 성공함을 확인.

## 전체 전투 화면 5종 사운드 통합 완료

서로 코드를 공유하지 않는 독립 전투 컴포넌트 5개를 전부 확인해서 사운드 통합함:

| 컴포넌트 | 용도 | 통합한 사운드 |
|---|---|---|
| `BattleScreen.jsx` | 스테이지 전투 | 공격/힐/버프/승리/레벨업 |
| `DungeonBattle.jsx` | 경험치·골드 던전, 무한의 탑 | 공격/힐/버프/승리/레벨업 |
| `JobDungeonBattle.jsx` | 전직 던전 | 공격/힐/버프/승리(전직 성공 시 레벨업 사운드도 함께) |
| `WorldBossBattle.jsx` | 월드보스 | 공격/힐/버프(공용 보스라 "처치" 개념 없어 승리 사운드 생략) |
| `PvPBattleScene.jsx` | PvP 연출(서버가 이미 판정한 결과를 재생) | 매 라운드 타격음 + 최종 승패 사운드 |

**부수 발견**: `DungeonBattle.jsx`(1차 커밋에서 사운드 누락됐던 걸 재검토 중 발견해 통합)에 코드를 다시 훑다가, 패배 메시지가 항상 "오늘 입장 횟수가 차감됐어요"로 고정돼있는 걸 발견 — 072에서 무한의 탑이 입장 무제한으로 바뀐 뒤에도 그대로 남아 사실과 다르게 표시되고 있었음. `dungeonEnemy.dungeonType === 'tower'`일 때 "같은 층을 다시 도전해보세요!"로 분기하도록 즉시 수정.

## 추가 통합: 업적/우편/합성

전투 화면 외에 손맛이 필요한 지점 3곳:

- 우편 수령/전체수령(`Mailbox.jsx`) — 골드 획득 사운드(전체수령은 1개 이상 성공했을 때만)

## 백그라운드 시 BGM 자동 정지 (사용자 피드백)

모바일에서 앱을 백그라운드로 보내거나 화면을 잠그면 `document.hidden`이 `true`가 되는데, 이때 `visibilitychange` 이벤트로 BGM을 자동 정지하고 복귀 시(BGM 설정이 켜져있는 경우만) 자동 재개함.

- `pauseBgmForVisibility()`/`resumeBgmForVisibility()`가 `settings.bgmEnabled`(사용자가 설정에서 켜고 끄는 값)는 절대 안 건드리고, `wasBgmPlayingBeforeHidden` 별도 플래그로만 "백그라운드 진입 직전 재생 중이었는지"를 기억함 — 설정에서 꺼놓은 상태로 백그라운드를 오가도 임의로 다시 켜지지 않음
- `App.jsx`에 `visibilitychange` 리스너 등록
- 헤드리스 브라우저로 두 시나리오(BGM 켜짐/꺼짐 상태의 백그라운드↔복귀) 모두 예외 없이 정상 실행됨을 확인
