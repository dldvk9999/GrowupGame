# BGM/효과음 (신규 콘텐츠, `lib/audio.js`)

## 개요

실제 음원 파일 없이 **Web Audio API의 오실레이터로 직접 합성**한 8비트풍 신스 사운드. 외부 에셋 로딩이 전혀 없어서 가볍고, 네트워크 상태와 무관하게 항상 작동함.

## 구성

- **효과음(SFX)**: 공격/힐/버프/승리/레벨업/골드/황금몬스터/신기록/뽑기결과(등급별 음색 차등) — 짧은 오실레이터 톤 조합
- **BGM**: 5음계(펜타토닉) 기반 짧은 루프 멜로디를 `setTimeout`으로 계속 이어붙여서 재생. 진짜 음원이 아니라 순차적으로 음을 합성해서 재생하는 방식이라 "찬찬한 신스 루프" 느낌에 가까움

## 브라우저 자동재생 정책 준수

브라우저는 사용자 상호작용 없이 오디오가 자동 재생되는 걸 막음. `App.jsx`가 `document`에 **1회성 클릭 리스너**를 등록해서, 화면 어디든 처음 한 번 클릭하면(BGM 설정이 켜져있는 경우) 그 클릭 이벤트 핸들러 안에서 `AudioContext`를 초기화/resume하고 BGM을 시작함 — 이후 리스너는 스스로 해제됨.

## 설정 (게임가이드 화면)

BGM on/off, 효과음 on/off, 볼륨 슬라이더를 설정 > 게임가이드 상단에 배치(테마 컬러 선택 UI와 같은 성격의 "개인화 설정" 묶음). `localStorage`에 저장, 서버 관여 없음.

## 통합 지점

- **전투(`BattleScreen.jsx`)**: 데미지/힐/버프 스킬 사용 시 각각 다른 효과음, 승리 시 승리 팡파레, 레벨업 이벤트가 있으면 약간의 딜레이 후 레벨업 사운드
- **뽑기(`SkillGacha.jsx`, `EquipmentGacha.jsx`)**: 결과 중 가장 높은 등급 기준으로 음색이 달라짐(등급 높을수록 화음이 더 풍부해짐)
- **황금 몬스터 이벤트**: 반짝이는 상승 아르페지오
- **무한의 탑 신기록**: 팡파레풍 사운드

## ⚠️ 배포 전 발견/수정한 버그: 첫 사운드 재생 실패

Playwright로 각 함수를 실제 헤드리스 브라우저에서 호출해 검증하던 중 발견: **모듈 로드 후 가장 처음 호출되는 사운드 함수 하나가 항상 에러를 내는 버그**가 있었음(`Failed to execute 'connect' on 'AudioNode': Overload resolution failed`).

**원인**: `sfx()` 함수가 `tone(freq, duration, type, sfxGain, peak, delay)`를 호출할 때, `sfxGain`은 **호출 시점에 이미 평가되어 고정된 값**으로 전달됨(JS의 함수 인자 평가 규칙). 그런데 `sfxGain`은 `tone()` 함수 *내부*에서 `getContext()`가 호출될 때 비로소 초기화됨 — 즉 최초 호출 시점엔 아직 `null`인 `sfxGain`이 그대로 `tone()`에 박제되어 전달되고, `tone()` 내부의 `getContext()` 호출로 전역 `sfxGain` 변수는 갱신되지만 이미 넘어온 매개변수는 여전히 `null`이라 `env.connect(null)`이 실패함. 두 번째 호출부터는 전역 `sfxGain`이 이미 초기화돼있어서 문제가 재현되지 않았음(그래서 테스트를 순차적으로 여러 함수 호출하는 방식이 아니었다면 놓쳤을 수도 있는 버그).

**수정**: `sfx()`에서 `tone()`을 호출하기 **전에** `getContext()`를 먼저 명시적으로 호출해서 `sfxGain`이 확실히 초기화된 뒤에 참조하도록 순서를 바꿈. `scheduleBgmNote()`는 이미 같은 순서(`getContext()` 먼저 → `bgmGain` 참조)로 작성되어 있어서 이 버그가 없었음(우연히 안전했던 패턴).

**검증**: 로컬 HTTP 서버 + Playwright 헤드리스 브라우저로 `audio.js`의 모든 export 함수(16개 시나리오: 효과음 10종 + 뽑기 등급별 5종 + BGM 시작/정지)를 실제로 호출해서 예외 없이 전부 성공함을 확인.

## DungeonBattle.jsx (던전/전직던전/무한의탑 공용)도 동일하게 통합

`BattleScreen.jsx`(스테이지 전투)와 `DungeonBattle.jsx`(던전·전직던전·무한의탑 공용, 완전히 별도 구현체)는 서로 코드를 공유하지 않는 독립 컴포넌트라, 사운드 작업 1차 커밋에서 `DungeonBattle.jsx`가 누락됐던 것을 재검토 중 발견하고 동일하게 통합함(데미지/힐/버프/승리/레벨업).

이 과정에서 함께 발견한 사소한 문구 버그도 같이 수정: 패배 메시지가 항상 "오늘 입장 횟수가 차감됐어요"로 고정되어 있었는데, 무한의 탑은 072에서 이미 입장 무제한으로 바뀌었으므로 이 문구가 더 이상 사실과 다름 — `dungeonEnemy.dungeonType === 'tower'`일 때는 "같은 층을 다시 도전해보세요!"로 분기하도록 수정함.

## 전체 전투 화면 5종 사운드 통합 완료

이 게임에는 서로 코드를 공유하지 않는 독립된 전투 컴포넌트가 5개 있는데(설계상 그때그때 복사+커스터마이징된 구조), 전부 확인해서 사운드를 통합함:

| 컴포넌트 | 용도 | 통합한 사운드 |
|---|---|---|
| `BattleScreen.jsx` | 스테이지 전투 | 공격/힐/버프/승리/레벨업 |
| `DungeonBattle.jsx` | 경험치·골드 던전, 무한의 탑 | 공격/힐/버프/승리/레벨업 (+패배 메시지 오류 수정, 아래 참고) |
| `JobDungeonBattle.jsx` | 전직 던전 | 공격/힐/버프/승리(+전직 성공은 레벨업 사운드도 함께) |
| `WorldBossBattle.jsx` | 월드보스 | 공격/힐/버프 (공용 보스라 "처치" 개념이 없어 승리 사운드는 생략) |
| `PvPBattleScene.jsx` | PvP 연출(서버가 이미 판정한 결과를 재생) | 매 라운드 타격음 + 최종 승패 사운드 |

## 부수 발견: 무한의 탑 패배 메시지 오류

`DungeonBattle.jsx`에 사운드를 추가하며 코드를 다시 훑다가 발견: 패배 메시지가 항상 "오늘 입장 횟수가 차감됐어요"로 고정돼 있었는데, 072에서 무한의 탑을 입장 무제한으로 바꾼 뒤에도 이 문구가 그대로 남아있어서 사실과 다르게 표시되고 있었음. `dungeonEnemy.dungeonType === 'tower'`일 때는 "같은 층을 다시 도전해보세요!"로 분기하도록 즉시 수정.

## 추가 통합: 업적/우편/합성

전투 화면 외에도 손맛이 필요한 지점 3곳에 추가로 통합:
- 업적 수령(`Achievements.jsx`) — 레벨업 사운드
- 우편 수령/전체수령(`Mailbox.jsx`) — 골드 획득 사운드(전체수령은 1개 이상 성공했을 때만)
- 장비 합성 성공(`Inventory.jsx`) — 버프 사운드

## 백그라운드 시 BGM 자동 정지 (사용자 피드백)

모바일에서 앱을 다른 앱으로 전환하거나 화면을 잠그면 `document.hidden`이 `true`가 되는데, 이때 `visibilitychange` 이벤트로 BGM을 자동으로 멈추고, 다시 앱으로 돌아오면(BGM 설정이 여전히 켜져있는 경우에만) 자동으로 재개함.

- `pauseBgmForVisibility()`/`resumeBgmForVisibility()`가 `settings.bgmEnabled`(사용자가 설정에서 명시적으로 켜고 끄는 값)는 절대 건드리지 않고, `wasBgmPlayingBeforeHidden`이라는 별도 플래그로만 "백그라운드 진입 직전에 실제로 재생 중이었는지"를 기억함 — 이러면 사용자가 설정에서 BGM을 꺼놓은 상태로 백그라운드를 오가도 절대 임의로 다시 켜지지 않음
- `App.jsx`에 `visibilitychange` 리스너를 등록해서 처리
- 헤드리스 브라우저로 두 시나리오(BGM 켜진 상태의 백그라운드↔복귀, BGM 꺼진 상태의 백그라운드↔복귀) 모두 예외 없이 정상 실행됨을 확인
