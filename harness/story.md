# 스토리 시스템 (신규 콘텐츠, 사용자 요청)

관련 파일: `lib/story.js`(오프닝), `lib/stageStory.js`(챕터 진입 스토리), `lib/storyArc.js`(메인 아크+인터루드), `components/StoryIntro.jsx`, `components/ChapterStory.jsx`, `components/StoryArtwork.jsx`, `components/StageSelect.jsx`

## 개요

기존엔 챕터 진입 스토리가 전부 **절차적 생성**(훅 문장 10개를 챕터 이름/속성과 조합)이라 10챕터마다 반복되고, 서사가 이어지지 않는다는 한계가 있었음(사용자 피드백: "더 박진감있고 긴장감있게, 탄탄한 스토리"). 100챕터 전부를 새로 쓰는 대신, **주요 분기점에만 진짜 서사를 배치**하고 나머지는 기존 절차적 훅을 유지하되 텍스트 풀을 확장(10종→20종)해서 반복 체감을 줄이는 절충안으로 설계.

## 메인 스토리 아크 (`lib/storyArc.js`) — "잊혀진 조율자"

기존 오프닝(`lib/story.js`)의 설정("세 기운이 원래 하나였지만 균형이 무너져 몬스터가 폭주한다")을 그대로 이어받아 배후 서사를 완성함:

- **진짜 배후**: 아주 오래전 세 기운을 처음 나눈 "태초의 조율자" — 나뉜 힘을 다시 하나로 합치려 폭주를 유발 중
- **라이벌 캐릭터**: "그림자 조련사" — 초반엔 적대적이지만(20챕터), 자신의 몬스터가 잠식당한 피해자임이 드러나고(50챕터), 봉인을 완전히 부수자는 상반된 신념으로 갈라섰다가(70챕터), 결말부엔 완전한 동료는 아니어도 같은 곳을 보는 관계로 수렴(95챕터)
- **12개 주요 비트**(1/10/20/30/40/50/60/70/80/90/95/100챕터)에 배치, `getStoryArcBeat(chapter)`가 있으면 `getChapterStory`가 절차적 훅보다 우선 사용
- 100챕터(최종장)도 이 아크의 일부로 포함 — 기존 "최후의 장" 범용 문구 대신 실제 결말(태초의 조율자와의 대면)로 교체

## 삽화 (`components/StoryArtwork.jsx`)

9종의 **재사용 가능한 상징적 SVG 무드아트**(여명/균열/그림자/폐허/폭풍/격돌/봉인/빛/옥좌) — 몬스터 스프라이트가 종별로 소수의 벡터를 재사용하는 것과 같은 철학. 상세한 인물 일러스트 대신 추상적 실루엣+색감 구도로 분위기만 전달해서, 외부 이미지 에셋/호스팅 없이 순수 SVG 코드로 구현(폰트/이미지 CDN 의존성 없음). 각 스토리 비트가 `imageKey`로 지정.

## 인터루드 (가끔 노출되는 추가 스토리)

스테이지(소스테이지) 이동마다 **12% 확률**로 짧은 추가 장면(텍스트+삽화)이 챕터 진입 스토리와 동일한 화면으로 뜸(`maybeGetInterlude()`, `App.jsx`의 `handleSelectStage`). 이미 진행 중인 챕터를 반복 방문해도 매번 뜰 수 있고, 6종 풀에서 무작위로 골라 보여줌 — 회상/단상 톤의 짧은 장면이라 매 챕터 최초진입 스토리와는 결이 다름.

- `App.jsx`가 `pendingStoryContent` state로 "챕터 최초진입 스토리"와 "인터루드"를 같은 `ChapterStory` 화면에서 통합 처리 — 어느 쪽이든 전투화면 진입 전에 뜨고, "계속하기"를 누르면 원래 가려던 스테이지로 그대로 이어짐(스테이지 이동 자체를 막지 않음)

## 스테이지 선택 화면에서 스토리 다시보기 (신규 콘텐츠, 사용자 요청)

`StageSelect.jsx`의 챕터 상세 패널에 "📖 스토리 보기" 버튼 추가 — 언제든 그 챕터의 스토리(주요 비트가 있으면 그걸, 없으면 절차적 스토리)를 **팝업**으로 다시 볼 수 있음. 삽화가 있으면 같이 표시됨. 기존 `modal-backdrop` 패턴을 재사용하되, 스토리 카드 특유의 그라디언트 배경(`.story-card`)을 유지하기 위해 `.story-popup-card`(max-height/overflow-y로 모바일 안전) 조합으로 구현. 각 챕터 카드에도 이미 스토리 첫 문장이 미리보기로 표시되고 있었음(기존 기능 유지).

## 알려진 범위

- 88개 챕터(주요 비트가 없는 챕터)는 여전히 절차적 생성 — 전부 손으로 쓴 서사로 교체하는 건 매우 큰 작업이라, 주요 전환점 위주로 진짜 서사를 배치하고 나머지는 "여정이 이어지는 중"이라는 자연스러운 여백으로 남겨둠
- 인터루드는 스테이지 선택→진입 흐름에서만 뜸(전투 중 실시간 이벤트 아님)
- 순수 클라이언트 콘텐츠라 서버 마이그레이션 없음
