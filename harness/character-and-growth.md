# 캐릭터/육성/전직

관련 파일: `growth.js`, `speciesData.js`, `speciesDbIds.js`, `jobAdvancement.js`, `monsters.js`

## 스타터/진화

- 스타터 3종: `fire_1`(이모탄) / `water_1`(아쿠파피) / `grass_1`(새프링)
- 각 속성은 3단계 진화(1→2→3차)가 있음: 예) 이모탄→이모드릴→이모라돈 (Lv.15, Lv.30에서 자동 진화)
- **진화**와 **전직**은 별개 시스템: 진화=외형/도감상 종족 변경, 전직=같은 종족 내 강함 단계
- 9종(3속성×3진화) 전부 벡터 SVG 일러스트 완성(`assets/sprites/FireStage1~3.jsx` 등)

### 능력치 상세 카드 (`MyPage.jsx`)

마이페이지에 "기본(레벨·전직) / 장비 보너스 / 스킬 보유효과 / 최종" 4행짜리 표로 ATK/DEF/HP 구성을 투명하게 보여줌. `App.jsx`가 이미 전투 화면들에 내려주던 `equipmentOnlyBonus`(용의 버프 배율이 안 낀 순수 장비 보너스)와 `skillPossessionAtk`를 그대로 재사용해서 별도 계산 없이 표시함. 용의 버프가 켜져있으면 표 아래에 "전투 시 20배 적용" 안내만 별도로 붙이고, 표 자체의 숫자는 배율 적용 전(순수 합산) 값으로 유지해서 "지금 순수하게 얼마나 강한지"와 "전투 중 실제로 얼마나 세지는지"를 헷갈리지 않게 분리함.

### 몬스터 애칭 (migration 058)

`owned_monsters.nickname` 컬럼은 001부터 존재했지만 클라이언트 어디서도 쓰인 적 없는 죽은 컬럼이었음(코드 재검토 중 발견). 마이페이지의 "대표 몬스터" 행 아래에 애칭 편집 UI를 추가해서 연결함.

- `set_monster_nickname(p_nickname)` RPC — 1~12자(한글/영문/숫자), `null`을 보내면 애칭 해제
- `hydrateMonster()`가 `nickname || speciesName`으로 표시 이름을 결정 — 애칭을 지으면 전투 화면 로그, 헤더 등 `player.name`을 쓰는 모든 곳에 자동으로 반영됨(종족 기본 이름 `speciesName`은 별도 필드로 항상 보존)
- 계정 닉네임(`profiles.nickname`, 평생 1회만 수정 가능)과는 완전히 별개 — 몬스터 애칭은 유일성 제약도 없고 자유롭게 여러 번 바꿀 수 있음(재미 요소라 제한을 둘 이유가 없음)

### 인기 속성 통계 (migration 068, 신규 콘텐츠)

도감 화면 하단에 "🌍 전체 유저 계약 속성 비율"을 막대그래프로 보여줌. `owned_monsters`가 "본인만 조회" RLS라 전체 집계는 `fetch_element_popularity()`(security definer)가 대신 계산 — 개인 식별정보 없이 속성별 인원수/비율(%)만 반환함. 활성 몬스터가 없는 유저는 집계에서 자동 제외됨(`is_active = true` 필터).

### 몬스터 도감 (`MonsterDex.jsx`)

마이페이지 상단에 있는 3×3(속성×진화단계) 그리드. 서버에 "발견 기록"을 저장하는 게 아니라, **매번 클라이언트에서 활성 몬스터의 현재 속성/단계를 기준으로 계산**함 — 내 속성과 같고 현재 도달한 단계 이하인 칸만 스프라이트가 보이고, 나머지는 실루엣(`?`)으로 잠김 표시. 이 게임은 스타터 1종만 계약해서 진화시키는 구조라 다른 두 속성은 항상 미발견 상태로 남는 게 정상 동작임(포획 기능이 생기기 전까지는). 발견된 칸에는 `speciesData.js`의 **레벨1 기준 기본 스탯**(⚔️공격 🛡️방어 ❤️체력)도 함께 보여줘서, 다음 진화 단계가 얼마나 강한지 미리 가늠할 수 있게 함(실제 성장/전직이 반영된 값이 아니라는 걸 안내문으로 명시). 별도 보상은 없는 순수 수집 열람 화면.

## 레벨업/스탯 성장

- **레벨업 경험치**: `expToNextLevel(level) = round(20 * level^1.5)`
- **스탯 성장**: `base스탯 × (1 + (level-1)*0.12) × 전직배율`
- `speciesKeyToDbId`/`dbIdToSpeciesKey`(`speciesDbIds.js`)로 클라이언트 문자열 키(`fire_1` 등)와 DB `species_id`(정수)를 매핑

## 전직 시스템 (1~5차)

- Lv.30/60/100/140/180에서 조건 충족 시 상단에 "✨ 전직 가능!" 배너가 뜨지만(`hasPendingJobAdvancement`), **레벨업만으로 자동 전직되지 않음** — 반드시 전직 던전(`stages-and-dungeons.md` 참고)을 클리어해야 실제 적용됨
- `owned_monsters.unlocked_job_tier`(0~5)가 "실제 적용된" 전직 단계의 단일 진실 공급원
- `jobAdvancement.js`가 두 개념을 분리해서 관리함:
  - `getEligibleTierNumber(element, level)`: 레벨 기준 "조건 충족" 단계 (알림용)
  - `getAppliedTier(element, unlockedJobTier)`: `unlocked_job_tier` 기준 "실제 적용" 단계 (스탯/스킬/외형에 반영되는 진짜 값)
- 전직할 때마다 **전용 스킬 1개씩 추가 습득**(누적, 기본 뽑기 스킬과 별개로 유지)
- 전직에 성공하면 **외형도 전용 그래픽으로 바뀜** (아래 "외형 변경" 참고)

### 전직 단계별 수치

| 단계 | 필요 레벨 | statMultiplier | 전용 스킬 배율 | 전용 스킬 쿨타임 |
|---|---|---|---|---|
| 1차 | 30 | 2.0배 | 3.6배 | 6.5초 |
| 2차 | 60 | 3.5배 | 4.6배 | 8초 |
| 3차 | 100 | 6.0배 | 6.2배 | 10초 |
| 4차 | 140 | 10.0배 | 8.2배 | 12.5초 |
| 5차 | 180 | 16.0배 | 10.6배 | 15.5초 |

- statMultiplier는 `growth.js`의 레벨 성장 보정에 **곱연산**으로 추가 적용됨
- 서버 `save_monster_growth` RPC의 스탯 상한선 공식도 이 최대값(16.0배)에 맞춰 매 단계 확장마다 함께 상향해왔음(migration 011/021/029)
- 전직 스킬은 전부 `damage` 타입, 속성별로 다른 이름/아이콘 사용 (`jobAdvancement.js`의 `JOB_TIERS` 참고)

### 외형 변경

- `getDisplaySpriteKey(speciesId, element, unlockedJobTier)`가 `unlockedJobTier > 0`이면 `${element}_job${tier}` 키를 반환해서, 진화단계 그림 대신 전직 전용 그림이 표시됨
- `JobTierSprite.jsx`(공용 렌더러)가 오라 크기/날개/왕관을 단계별로 화려하게 렌더링 — tier가 오를수록 오라 반지름이 커지고(tier≥2에서 날개, tier≥3에서 왕관+할로 추가), **4~5차도 별도 아트 없이 기존 컴포넌트가 tier 값만 받아서 자동으로 더 화려하게 확장됨**(오라만 더 커지고 3차와 동일한 왕관/할로 장식 유지)
- `assets/sprites/index.jsx`에 3속성×5단계 = 15개 스프라이트 키 전부 등록됨

## 전직 던전과의 관계

- 전직 던전 클리어(`claim_job_dungeon` RPC)가 `owned_monsters.unlocked_job_tier`를 실제로 올리는 유일한 경로
- 자세한 던전 난이도/보스 스탯/입장 규칙은 [`stages-and-dungeons.md`](./stages-and-dungeons.md)의 "전직 던전" 섹션 참고

### 캐릭터 카드 요약 (신규 콘텐츠, `MyPage.jsx`)

마이페이지 상단에 닉네임+칭호, 대표 몬스터, 그리고 4개 핵심 지표(PvP 티어/무한의 탑 최고층/업적 개수/클리어 스테이지)를 한눈에 보여주는 요약 카드 추가. 기존에 흩어져 있던 정보를 한 곳에 모아서 "내 캐릭터가 지금 어디까지 왔는지" 바로 파악 가능하게 함.

- 순수 프레젠테이션 콘텐츠 — 새 서버 호출은 업적 개수(`fetchClaimedAchievements`, 이미 있는 함수 재사용) 하나뿐, 나머지는 전부 이미 로드된 데이터 조합
- 2열 그리드 레이아웃이라 항목 개수가 늘어나도 모바일에서 안전(그리드는 flex row보다 넘침 위험이 낮음), 그래도 극단값(다이아몬드+9999층+29/29+1000/1000)으로 Playwright 실측 검증 완료

### 성장 곡선 미리보기 (신규 콘텐츠, `MyPage.jsx`)

마이페이지 능력치 상세 카드 아래에 "지금 전직 단계 기준으로 레벨만 올랐을 때"의 예상 스탯(현재/+10/+30/+50레벨)을 보여주는 표 추가. `lib/growth.js`의 `scaleStats(species, level, unlockedJobTier)` 함수를 그대로 재사용해서 계산(이미 실제 레벨업 처리에 쓰이는 검증된 공식이라 별도 로직 중복 없음) — 순수 클라이언트 계산, 서버 호출 없음.

- 장비/스킬 보너스는 의도적으로 미포함(순수 종족 성장곡선만 보여주기 위함)이라는 걸 안내문에 명시
- 미래 진화/전직은 예측 불가능하므로 "현재 전직 단계 기준"이라는 전제를 명확히 표기해서 오해 방지
- 큰 숫자(백만 단위) 조합으로 모바일 렌더링 시 넘침 없음을 Playwright로 재검증
- **전투력 컬럼 추가**(사용자 경험 개선) — ATK/DEF/HP 뒤에 각 레벨 시점의 예상 전투력(`calculateCombatPower`로 계산)도 함께 표시해서, 스탯 숫자만으로는 체감이 어려운 성장폭을 한 눈에 비교할 수 있게 함. 5개 컬럼 극단값 조합으로도 모바일 넘침 없음 재검증 완료

### 다음 목표 위젯 (신규 콘텐츠, `lib/nextGoals.js`)

캐릭터 카드 하단에 "🎯 다음 목표" 칩 리스트 추가 — PvP 다음 티어까지 남은 승수, 무한의 탑 다음 10층 마일스톤까지 남은 층수, 다음 출석 마일스톤(7/30/100/200)까지 남은 횟수, 코스튬 컬렉션 완성까지 남은 종수(20종 기준), **전투력 마일스톤(10K/100K/1M, 083 업적과 연동)까지 남은 수치**를 한 곳에 모아서 보여줌. 여러 시스템에 흩어진 "다음 목표"를 순수 클라이언트 계산으로 통합(서버 호출 없음, 이미 로드된 데이터만 조합) — `pvpTier.js`의 `getWinsToNextTier` 등 기존 헬퍼를 재사용. 전투력은 `MyPage.jsx`의 스탯 상세 카드가 이미 계산해두는 최종 atk/def/hp(장비+스킬 보유효과 포함)를 그대로 `calculateCombatPower`에 넣어서 산출 — 서버의 `fetch_my_combat_power()`와 정확히 같은 값은 아닐 수 있지만(스킬 보유효과 계산 방식 차이 가능성), 순수 참고용 위젯이라 문제없음. 5개 칩이 동시에 표시되는 경우도 모바일 실측 검증 완료(넘침 없음).

### 연속 접속 스트릭 표시 (신규 콘텐츠, `lib/loginStreak.js`)

캐릭터 카드에 "🔥N일 연속 접속" 배지 표시(2일 이상일 때만). **순수 클라이언트 동기부여용 표시**이고 어떤 보상 계산에도 쓰이지 않음 — `localStorage`로 "이 기기에서 어제도 접속했는지"만 비교해서 스트릭을 이어가거나 리셋함. 서버 검증 없는 값이라 기기를 바꾸거나 브라우저 데이터를 지우면 리셋되는데, 애초에 보상과 무관한 재미 요소라 문제 없음(만약 서버 검증이 필요한 보상형 스트릭이었다면 훨씬 신중하게 설계해야 했겠지만, 이건 순수 표시용이라 안전).

### 다음 레벨까지 예상 소요시간 (신규 콘텐츠, `lib/idleTimeEstimate.js`)

성장 곡선 미리보기 카드에 "⏱️ 자동사냥만으로 다음 레벨까지 약 N분/시간" 추가 — 자동사냥 경험치 공식(`stages.js`의 `getIdleMonster`와 동일 공식, 최근 하향된 0.19 배율 반영)과 처치 텀(1.5초)을 그대로 재사용해서 순수 클라이언트로 추정함. 챕터는 클리어한 스테이지 수(`clearedCount / 10`)로 근사(정확한 진행 챕터가 아니라 추정치). 스테이지 도전/던전 등을 병행하면 더 빨라진다는 점을 문구에 명시해서 "이게 최소 하한선"이라는 걸 오해 없이 전달함. 시뮬레이션 결과 레벨5~220 전 구간에서 2분~10분 수준으로 합리적인 범위임을 확인, 모바일 줄바꿈도 검증 완료.

**던전 깊이 마일스톤도 추가됨**(088 업적과 연동) — exp/gold 던전 최고 진행층 기준으로 다음 마일스톤(100/300/500)까지 남은 층수 표시. 6개 칩 동시표시 모바일 재검증 완료.

**스킬 컬렉션/최고 강화수치 목표도 추가됨**(092/091 업적과 연동) — 각각 남은 스킬 종수(50종 기준), 남은 강화수치(+1000 기준)를 표시. 신규 유저가 아무 목표도 안 채웠을 때 나올 수 있는 최악의 경우(8개 칩 동시표시)까지 모바일 실측 검증 완료(넘침 없음).
