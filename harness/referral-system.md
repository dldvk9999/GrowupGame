# 친구 추천 시스템 (migration 065, 신규 콘텐츠)

관련 파일: `MyPage.jsx`, `lib/auth.js`(`setReferrer`), migration 065

## 개요

가입 후 24시간 이내에 딱 1번, 다른 유저의 닉네임을 "추천인"으로 등록할 수 있음. 추천받은 유저가 레벨 10 업적을 달성하면(실제 플레이 시작 신호) 추천인에게 골드 2000 보너스 우편이 감.

`handle_new_user` 트리거(001, 모든 신규 가입자가 거치는 핵심 지점)는 건드리지 않고, 가입 완료 후 별도 RPC(`set_referrer`)로 처리해서 회원가입 안정성에 영향이 없게 설계함.

## 악용 방지 장치

1. **가입 후 24시간 이내에만 설정 가능** — 오래된 계정의 뒤늦은 추천인 등록 방지
2. **한 번 설정하면 변경 불가** — `referred_by is not null`이면 재시도 거부
3. **자기 자신을 추천인으로 못 넣음**
4. **보너스는 "실제로 레벨10을 찍었을 때"만 지급** — 단순 가입만으로는 안 나감(`claim_achievement`의 `level_10` 케이스 안에서 처리)
5. **`mails.source_key`(`referral_bonus_유저id`) 유니크 제약**으로 중복 지급 방지

⚠️ **알려진 한계**: 완전한 멀티 계정 어뷰징(같은 사람이 여러 이메일로 가입해서 서로 추천) 방지는 이메일 인증 강화 등 이 프로젝트 범위 밖의 인프라가 필요해 근본적으로 막을 수 없음. 다만 보상이 작아(2000골드) 어뷰징 실효성이 낮음 — 완전 방지보다는 "적당히 억제" 수준으로 설계.

## 클라이언트

- `MyPage.jsx`에 "🤝 친구 추천인 등록" 카드 — `profile.created_at` 기준 24시간 경과 여부를 클라이언트에서도 미리 계산해 조건 안 맞으면 폼을 숨김(서버가 최종 검증은 항상 다시 함, 순수 UX 편의). 이미 등록돼있으면 안내 문구만 표시
- `set_referrer(p_referrer_nickname)` 실패 시(잘못된 닉네임, 24시간 경과, 이미 등록됨 등) 서버가 명확한 한국어 에러 반환

## claim_achievement 재정의 (065)

053/059/060/061에서 계속 재정의된 `claim_achievement`에 "레벨10 업적 달성 시 추천인 보너스 지급" 로직만 추가. 반환타입(`integer`) 그대로라 DROP FUNCTION 불필요 — 배포 전 24개 업적 CASE 전부가 이전 버전(061)과 diff로 일치함을 확인함.

## 내 추천 현황 표시

마이페이지에 "내가 추천한 친구" 수 표시(`fetchMyReferralCount`) — `profiles.referred_by = 내ID` 행 개수만 세는 것이라, `profiles`가 이미 "누구나 조회 가능" RLS라 별도 RPC 없이 직접 count 쿼리로 조회함(랭킹/월드보스 순위 조회와 동일한 패턴). 1명 이상 추천했을 때만 행이 보임.

## 내 닉네임 복사 버튼 (신규 콘텐츠)

캐릭터 카드의 닉네임 옆에 📋 복사 버튼 추가 — 클릭하면 `navigator.clipboard.writeText()`로 복사하고 2초간 ✅로 바뀌어 성공 피드백(클립보드 API 실패 시 토스트로 닉네임을 직접 보여줌). 친구 추천 시스템(065)이 "닉네임을 직접 입력"해야 하는 구조라, 자신의 닉네임을 친구에게 쉽게 공유하도록 돕는 편의 기능.

⚠️ **개선(신규 콘텐츠)**: 처음엔 닉네임 텍스트만 복사했는데, 받는 사람 입장에선 "이게 뭐지?" 할 수 있는 맥락 없는 문자열이라 개선함. 지금은 "키우기 게임 같이 해요! 마이페이지 > 친구 추천인 등록에서 제 닉네임 'OO' 입력해주세요 🎮" 형태의 **초대 방법까지 포함된 완성된 문구**를 복사함 — 붙여넣기만 해도 바로 전송 가능해서 실제 추천 전환율에 도움이 될 것으로 기대. 버튼 툴팁도 "친구 초대 문구 복사"로 갱신.
