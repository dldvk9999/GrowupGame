# 친구 추천 시스템 (migration 065, 신규 콘텐츠)

관련 파일: `MyPage.jsx`, `lib/auth.js`(`setReferrer`), migration 065

## 개요

가입 후 24시간 이내에 딱 1번, 다른 유저의 닉네임을 "추천인"으로 등록할 수 있음. 추천받은 유저가 레벨 10 업적을 달성하면(=실제로 플레이를 시작했다는 신호) 추천인에게 골드 2000 보너스 우편이 감.

`handle_new_user` 트리거(001, 모든 신규 가입자가 거치는 핵심 지점)는 건드리지 않고, 가입 완료 후 별도 RPC(`set_referrer`)로 처리해서 회원가입 자체의 안정성에 영향이 없게 설계함.

## 악용 방지 장치

1. **가입 후 24시간 이내에만 설정 가능** — 오래된 계정이 뒤늦게 추천인을 붙이는 것 방지
2. **한 번 설정하면 변경 불가** — `referred_by is not null`이면 재시도 거부(재시도 남용 방지)
3. **자기 자신을 추천인으로 못 넣음**
4. **추천 보너스는 "실제로 레벨10을 찍었을 때"만 지급** — 단순 가입만으로는 안 나감(`claim_achievement`의 `level_10` 케이스 안에서 처리)
5. **`mails.source_key`(`referral_bonus_유저id`) 유니크 제약**으로 중복 지급 방지

⚠️ **알려진 한계**: 완전한 멀티 계정 어뷰징 방지(같은 사람이 여러 이메일로 가입해서 서로 추천)는 이메일 인증 강화 등 이 프로젝트 범위 밖의 인프라가 필요해서 근본적으로 막을 수 없음. 다만 보상 자체가 크지 않아(2000골드, 스테이지 클리어 보상 대비 미미한 수준) 여러 계정을 만드는 수고 대비 이득이 적어 실효성 낮은 어뷰징으로 설계함 — 완전한 방지보다는 "적당히 억제"하는 수준.

## 클라이언트

- `MyPage.jsx`에 "🤝 친구 추천인 등록" 카드 — `profile.created_at` 기준 24시간 경과 여부를 클라이언트에서도 미리 계산해서 조건 안 맞으면 폼 자체를 숨김(서버가 최종 검증은 항상 다시 하므로, 이건 순수 UX 편의). 이미 추천인이 등록돼있으면 안내 문구만 표시
- `set_referrer(p_referrer_nickname)` 실패 시(잘못된 닉네임, 24시간 경과, 이미 등록됨 등) 서버가 명확한 한국어 에러 메시지를 반환

## claim_achievement 재정의 (065)

053/059/060/061에서 계속 재정의되어 온 `claim_achievement`에 "레벨10 업적 달성 시 추천인 보너스 지급" 로직만 추가. 반환타입(`integer`)은 그대로라 DROP FUNCTION 불필요 — 배포 전 24개 업적 CASE 전부가 이전 버전(061)과 diff로 정확히 일치함을 확인함.

## 내 추천 현황 표시

마이페이지에 "내가 추천한 친구" 수를 보여줌(`fetchMyReferralCount`) — `profiles.referred_by = 내ID`인 행 개수를 세는 것뿐이라, `profiles` 테이블이 이미 "누구나 조회 가능" RLS라 별도 RPC 없이 클라이언트가 직접 count 쿼리로 조회함(랭킹/월드보스 순위 조회와 동일한 패턴). 1명 이상 추천했을 때만 행이 보임.

## 내 닉네임 복사 버튼 (신규 콘텐츠)

캐릭터 카드의 닉네임 옆에 📋 복사 버튼 추가 — 클릭하면 `navigator.clipboard.writeText()`로 닉네임을 클립보드에 복사하고, 2초간 ✅로 바뀌어 성공 피드백을 줌(클립보드 API 실패 시엔 토스트로 닉네임을 직접 보여줌). 친구 추천 시스템(065)이 "닉네임을 직접 입력"해야 하는 구조라, 자신의 닉네임을 친구에게 쉽게 공유할 수 있게 돕는 편의 기능.
